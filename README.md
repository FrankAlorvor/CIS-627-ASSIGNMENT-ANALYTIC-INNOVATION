# Analytics Innovation & Version Control – Sports Decision Support Project

## Project Overview
This project presents a sports analytics tool called the **Player Rotation Readiness Dashboard**. The goal is to help coaches and team managers decide whether a player should get more playing time, stay in the same role, or have reduced minutes. It combines performance and workload data into one simple dashboard for easy decision-making.

## Decision-Making Problem
Sports teams often struggle with deciding how to manage player rotation and playing time throughout a season. These decisions affect team performance, player development, and the risk of fatigue or injury. This dashboard helps improve those decisions by providing clear insights on whether a player’s minutes should increase, decrease, or stay the same.

## Proposed Analytics Approach
The dashboard uses data such as:
- Minutes played  
- Game frequency  
- Shooting efficiency  
- Scoring trends  
- Turnover rates  

It analyzes how workload and performance are related over time. Instead of using just one metric, it combines multiple indicators to give a more complete and balanced view of player readiness.

## Use by Decision Makers
- **Coaches** can use the dashboard before games or during weekly planning to adjust player minutes.  
- **Team executives** can use it to monitor player usage over the season and support roster decisions.  

The focus is on making the data simple, clear, and useful for real decisions.

PROTOTYPE
## Prototype Enhancement

This update improves the Player Rotation Readiness Dashboard by adding a **visual classification layer**. Instead of only displaying raw performance and workload data, the dashboard groups players into clear categories such as:
- **Ready for Expanded Role**
- **Stable Current Role**
- **Monitor for Fatigue**

This enhancement makes decision-making faster and more practical. Coaches and team executives often need quick, clear insights rather than detailed tables of numbers. By turning multiple data points into simple status labels, the dashboard becomes easier to read and more useful for game preparation and planning.

## Prototype Evaluation

The enhancement should be added to the main project if it makes the dashboard easier to use and helps improve decisions about player rotation. In analytics projects, success depends not just on having a good idea, but also on whether decision makers find the tool simple and practical to use.

Feedback should mainly come from users like coaches, performance staff, and front office leaders. They should evaluate whether the update:
- Makes information clearer  
- Fits well into their daily workflow  
- Builds confidence in playing-time decisions  

If the enhancement improves usability without adding confusion, it should be included in the final version.
